# TypeScript 进阶学习计划（资深 JS 开发者专用）

> 受众：已掌握 JS 执行上下文、闭包、原型链、事件循环、异步编程的开发者
> 目标：8 周内掌握 TS 类型系统全貌，达到能写库、设计类型 API、看懂任意 d.ts 的水平
> 学习哲学：**类型系统本身就是一门函数式语言**——把类型当代码写，而不是当注解写

---

## 心智模型转换（开始前必读）

作为资深 JS 开发者，你需要建立的核心认知：

| JS 思维 | TS 思维 |
|---|---|
| 值有类型，运行时检查 | 类型在编译时存在，运行时被擦除 |
| 鸭子类型（duck typing） | 结构化类型（structural typing） |
| 函数签名是约定 | 函数签名是契约，编译器强制 |
| 灵活就是力量 | 精确就是力量；any 是失败 |
| 一个变量可以变类型 | 类型收窄（narrowing）是核心技巧 |

**关键陷阱**：TS 的类型系统是图灵完备的，可以做出非常复杂的推导。不要把它当作"加注解的 JS"——把它当作 **"在类型层面写程序"** 才能真正掌握。

## 学习方法论

### 1. 三遍法
每个阶段建议过三遍：
- **第一遍：通读概念** — 建立全貌，不抠细节
- **第二遍：动手做练习** — 在 [TS Playground](https://www.typescriptlang.org/play) 或本地写
- **第三遍：复现 + 教学** — 不看资料把核心概念讲一遍（可以写笔记或录音）

### 2. Playground 优先
**永远先用 TS Playground 跑一下**。它能让你立刻看到：
- 推导出的类型（hover 变量名）
- 编译后的 JS（看类型如何被擦除）
- 错误信息的完整位置

### 3. 错误信息阅读法
TS 报错以"难读"著称。养成习惯：
1. 看错误的**最后一行**（最具体的问题）
2. 看 "Type X is not assignable to Type Y" 中 X 和 Y 的差异
3. 用鼠标 hover 出问题的变量，对比期望类型

### 4. 不要轻易 `any` / `as`
- `any` 关闭了类型检查 → 等于回到 JS
- `as` 强转 → 你在告诉编译器"相信我"，但你可能错了
- 优先使用 `unknown` + 类型守卫
- 优先用泛型而不是 any

### 5. 配套练习仓库
本目录的 `exercises/` 下提供了与各阶段配套的练习起步模板。每个阶段练习完，把答案 commit 一次，方便回看进步。

## 进阶里程碑

### 60 分（W1-W3 后）
- 能给任意 JS 函数加正确的类型注解
- 看得懂 `lib.es5.d.ts` 中常用 API 的类型定义
- 知道何时用 interface 何时用 type

### 80 分（W4-W5 后）
- 能写出带泛型约束的工具函数（如 `pick`、`get`）
- 能用可辨识联合处理状态机
- 能用 `keyof`、`typeof`、`in` 做类型派生

### 95 分（W6-W7 后）
- 能独立实现 `Partial`、`ReturnType`、`Awaited` 等内置工具类型
- 能用条件类型 + `infer` 做类型推断
- 能解出 type-challenges medium 题
- 看到 `T extends infer U ? ... : never` 不再发懵

### 满分（W8 后）
- 能为大型项目设计 tsconfig 和模块边界
- 能为 React 组件设计精确的 props 类型（含泛型组件、polymorphic 组件）
- 能识别 TS 编译性能瓶颈并优化
- 能阅读 lodash、zod、tRPC 等库的类型定义并理解设计意图

## 学习节奏建议

- **每天 1-1.5 小时**：30 分钟读概念 + 60 分钟练习
- **周末 2-3 小时**：完成阶段练习 + 整理笔记
- **不要跳阶段**：泛型没吃透就上类型体操会卡很久
- **看不懂就标记 ❓ 继续走**：很多概念在后面阶段会自然回扣
